1시작하기

택인택, 어떤 서비스인가요?

  • 서비스명에서 유추할 수 있듯이 택 안에 또 다른 택이 있는 서비스입니다.
  • 원하는 상품을 기존 소비자가에서 단 1%의 가격으로도 구매할 수 있는 기회를 실현시킨 서비스입니다.
  • 서비스 이용시 단순히 비용절약 뿐만 아니라 기대치 못했던 행운을 얻는 재미까지 즐길 수 있습니다.
  • 각박한 현실을 살아가는 현대인들에게 택인택은 ‘재미’와 ‘희망’을 통하여 ‘행복’을 나누어주는 서비스가 될 것입니다.
“구매 순간마다 다양한 가격에 상품을 판매하는 방법”
대한민국 : 특허 제10-2015-0127427
PCT국제특허 : PCT/KR2016/009839
2까보기

택 속에 숨겨진 또 하나의 택, 당신만의 행운가를 확인하는 방법

  • 택인택에서의 ‘까보기’란, 택 안에 또 다른 택(가격)을 확인하는 일련의 과정입니다.
  • 택인택에서 판매하는 하나의 상품에는 여러가지 택(가격)이 존재하며, 까보기 버튼을 클릭하여 자신이 뽑은 택의 가격(할인율)을 확인 하실 수 있습니다.(최대 99% 할인)
  • 택(가격) 구성 확인
    구매를 원하는 상품이 정해졌다면,
    'Tag구성'을 보고 상품의 할인율 및 수량 분포도를 확인
  • 까보기 버튼 클릭
    상단의 있는 '까보기' 버튼을 클릭하면 → (Start)
  • 행운의 숫자 선택
    내가 원하는 행운의 숫자를 선택
    (직접입력 또는 자동선택 중 택일)
  • 오픈과정(특허 10-2017-0013756)
    - Start! : 출발지점으로 최근(직전) 오픈한 다른 회원의 도착지점
  • 오픈과정(특허 10-2017-0013756)
    - Input : 출발지점에서 선택한 숫자만큼 이동하는 과정(구매완료, 오픈중 인 택은 건너뜀)
  • 오픈과정(특허 10-2017-0013756)
    - Result : 내가 받은 택
  • 내가 받은 택(GOAL)의 가격(할인율) 확인
    - 내가 입력한 행운의 숫자를 기반으로 선택된 택(TAG)의 가격(할인율) 확인
    - 내가 받은 택(TAG)의 번호는 '까보기'를 선택한 다음 회원의 시작번호로 쓰임
  • 택(TAG) 사용하기
    - 확인한 택(TAG)으로 주문 / 양도 / 한번더 선택
    ※ 건전한 시스템 운영을 위해 60초 이내에 주문 / 양도 / 한번 더 중 택일
3가격구성

극한의 할인가, 어떻게 구성되어 있을까요?

  • 택인택 상품에는 각각 다른 할인율이 적힌 가격 택이 100 ~ 9999개가 있습니다. (상품별 구성)
  • 최대 할인율인 99%부터 단계별로 할인구간에 할당된 할인율 및 수량은 그래프에 표기된 숫자를 참조하세요.
4즉시구매

택 오픈 없이도, 싼 값에 급매물을 구매할 수 있습니다.

  • 택오픈(까보기)과 상관없이 난 이정도 가격이라면 OK~라고 한다면,
    즉시 구매 대기자 등록을 통하여 즉시구매 희망가격을 제시하고 기다리면 됩니다.
  • 제시한 가격조건에 맞는 판매자가 양도를 하면 즉시 자동거래가 이루어집니다.
  • 제시한 즉시구매 가격이 높을수록 구매 대기자 리스트에서 상단에 노출됩니다.
5돈버는 쇼핑몰(국내최초)

택 오픈만으로도 수익을 올릴 수 있는 획기적인 서비스

  • 까보기를 통해 얻은 Tag의 권리를 즉시 구입을 희망하는 구매자에게 양도하여 수익을 발생 시킬 수 있습니다.
  • 예) A상품의 판매가 10,000원 / 까보기비용 100원 / 내가 받은 Tag의 할인율(99%→100원에 구매가능) / 즉시구매가 9,000원 =
    9,000원 - (100원+100원) = 8,800원 수익(수수료차감전)
  • 99%할인된 Tag을 뽑은 고객은 수익을 얻고, 즉시구매자는 시중가보다 저렴하게 원하는 상품을 즉시 구입할 수 있습니다.
6공정성

공정한 기회와 철저한 보안을 위한 택인택의 노력

  • 누가 봐도 공정한 추첨 방식은 택인택이 꼭 해결해야 했던, 난제 중에 난제였습니다.
  • 또한 내부자 및 시스템 개발자의 접근을 원천적으로 막아야 했습니다.
  • 이를 통하여 택인택 모든 회원에게 공정한 기회와 행운을 제공하고, 택 오픈과정에 조작이 개입할 수 없도록 만반의 조치를 취하고 있습니다.
  • 1.
    제품에 대한 할인율과 택 수량이 정해지면 보안 서버의 난수 발생 시스템에 의해 택이 생성되고 렌덤으로 배열됩니다.
  • 2.
    생성된 택은, 택 오픈 전에 그 할인율을 확인할 수 없으며, 만약 택을 오픈한 흔적이 있다면 사전에 이를 검열할 수 있습니다.
  • 3.
    서비스 운영서버와 택 생성서버를 각기 다른 기간과 각기 다른 장소에서 각기 다른 담당자가 운영하여 동일인이 동시에 두 시스템을 컨트롤 할 수 없도록 원천 차단하였습니다.
  • 4.
    행운의 숫자 카운팅 시작 지점, 즉 START 지점을 사전에 알 수 없도록 하여 최종 도착지점(GOAL)을 유추할 수 없도록 하였습니다.
  • 5.
    택 카운팅(spinning)시 구매 완료된 택이나 오픈 중인 택은 제외시켜 도착지점(GOAL)을 더욱더 유추할 수 없도록 하였습니다.
  • 6.
    최초 택오픈시나, 일정 인원 미만이 동시에 택을 오픈하는 경우 지속적으로 로봇을 개입시켜 START 지점을 수시로 변동시켜 START 지점을 사전에 알 수 없도록 하였습니다.
  • 설령 99% 할인된 택이 몇 번 택에 있는지 사전에 안다고 해도 상기와 같은 시스템의 운영과 변수입력으로 인하여 도착지점을 유추할 수 없도록 하였습니다.
  • 택인택은 정직한 운영과 공명정대한 택 오픈을 위해 대한민국 NO.1 추첨시스템을 개발하였습니다.
